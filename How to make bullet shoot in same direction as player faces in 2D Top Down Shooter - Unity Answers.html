    Antwort von robertbu · 27 04 2014 um 23:02

    Assuming this script is on the character, you can use this transform to find the direction. First, identify forward for your character when the rotation is (0,0,0). For most 2D game it is either 'transform.up' or 'transform.right'. Then you can change line 33 to:

    1.     bPrefab.rigidbody2D.AddForce(transform.up * bulletSpeed);

    Or use 'transform.right' if the right of your character is forward.

    Assuming the above code is on the object that rotates, and that everything here is sprites, then first you need to change line 31 to:

      Rigidbody2D bPrefab = Instantiate(bulletPrefab, new Vector3(transform.position.x + xValue, transform.position.y + yValue, transform.position.z), transform.rotation) as Rigidbody2D;

    Note I change the last parameter of the Instantiate() to 'transform.rotation'. This is what I meant when I said, " can use this transform" in the original answer.

    Next you need to match the rotation of the player sprite to the bullet prefab. Based on your comment that is up. In other words, when your player sprite has a rotation of (0,0,0), the direction you want to fire the bullet is up. You also need to make the bullet prefab point up when the rotation is 0.0,0.

    Finally, the AddForce() should be the same as I indicated in my original answer:

      bPrefab.rigidbody2D.AddForce(transform.up * bulletSpeed);

    Antwort von Winterblood · 27 04 2014 um 23:06

    Assuming the PlayerShooting script is attached to your player GameObject...set the force on your prefab bullet to use the player's "transform.up" instead of the global "Vector2.up". (Or you might need to use "transform.right" depending how you've oriented your sprite.)
